 Why a spreadsheet

Using a spreadsheet to create the tier list allows you to keep the file for later editing. It also makes for a better user interface than just a text file.

I'm using LibreOffice 7.1.

 Tiers

Each tier is represented by one sheet of the tiers.ods spreadsheet. A tier consists of a header and a list of entries. Both headers and entries are images.

 Image source

The images need to be hosted somewhere. Currently there are two options.

Direct URL

Upload your images to an image hosting service like Postimages and use the direct URL.

Google Drive

Upload your images to Google Drive.


  1. Create a folder for the images.

  2. Make the folder public ("Anyone on the internet with this link can view").

  3. Get a share link for each image ("Get link").

Note on image size

It's best for all the images to be the same size. MAL's BBCode doesn't allow for resizing, so the desired image size needs to be set before upload. Remember to make the header width divisible by the entry image's width if you want the images to tile nicely.

 Headers

The tiers.ods file has links to some default headers uploaded to Postimages. Each tier has a color based on the typical tier list style. They don't contain any text, however. I generated these headers from this mal_tiers_slim.xcf file (GIMP format). If you wish to keep the same tier list color scheme, you can use it to create your own customized headers.

 Settings

In the SETTINGS sheet you can specify the order in which the tiers will be displayed. You can also remove a tier from the "Tier order" list to exclude it from the final tier list. The "Entries per row" setting decides how many entries will be displayed in one row.
